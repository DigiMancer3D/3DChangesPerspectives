*This should work on **Kubuntu 24.04**, **generic Linux 24.xx (Ubuntu/Debian-based or any modern distro with apt)**, and **generic Windows 10/11**. No on-chain activations, soft forks, or protocol changes being required.*

---

**Important Realities Before You Start**
- QSB is **experimental** (April 2026 release). The pinning + digest search is proven to work; full end-to-end spending is usable but requires manual miner submission (e.g., Slipstream).
- Cost: ~$75–$150 in GPU compute per high-value spend (embarrassingly parallel; scales with more GPUs).
- Hardware: NVIDIA GPU with CUDA support is **strongly recommended** (RTX 40-series or better). CPU-only is theoretically possible but impractically slow.
- Transactions are **non-standard** (bare script, large witness) → cannot be relayed via normal P2P; you must submit directly to a miner.
- You will still use **btcd** as your full node and **btcwallet** (or a simple Go wrapper) for normal operations. QSB only handles the quantum-safe *spend* creation.

### Phase 0: Common Prerequisites (All Platforms)
1. Have ~0.01+ BTC in a **legacy P2PKH or P2SH address** you control (QSB uses bare legacy scripts).
2. Run **btcd** with RPC enabled (`-rpcuser`, `-rpcpass`, `-txindex=1` recommended).
3. NVIDIA drivers + CUDA Toolkit 12.x (latest stable).
4. Git, Python 3.10+, Go 1.21+.

### Phase 1: Install btcd + btcwallet (Node + Wallet Base)

**Kubuntu 24.04 / Generic Linux**
```bash
# 1. Install Go (latest)
sudo apt update && sudo apt install -y golang-go git make build-essential
go version  # should be 1.21+

# 2. Install btcd + btcwallet
go install -v github.com/btcsuite/btcd/...@latest
go install -v github.com/btcsuite/btcwallet/...@latest

# 3. Add to PATH (add to ~/.bashrc)
echo 'export PATH=$PATH:$(go env GOPATH)/bin' >> ~/.bashrc
source ~/.bashrc

# 4. Start btcd (create config first)
mkdir -p ~/.btcd
cat > ~/.btcd/btcd.conf << EOF
rpcuser=youruser
rpcpass=yourstrongpass123
txindex=1
EOF
btcd --config ~/.btcd/btcd.conf
```

**Windows (10/11)**
1. Download the latest **MSI installers** from:
   https://github.com/btcsuite/btcd/releases
   https://github.com/btcsuite/btcwallet/releases
2. Run both MSIs (they install to `C:\Program Files\btcd` etc.).
3. Add to PATH: `C:\Program Files\btcd` and `C:\Program Files\btcwallet`.
4. Create `btcd.conf` in `%APPDATA%\btcd\` with the same rpcuser/rpcpass/txindex as above.
5. Launch `btcd.exe` from Start Menu or PowerShell.

### Phase 2: Install QSB (Quantum-Safe Pipeline + GPU Search)

**Kubuntu 24.04 / Generic Linux (Native – Recommended)**
```bash
# 1. Install system deps
sudo apt update
sudo apt install -y git python3 python3-pip python3-venv libssl-dev make cuda-toolkit-12-6  # or latest CUDA

# 2. NVIDIA drivers (if not already installed)
sudo ubuntu-drivers autoinstall

# 3. Clone QSB repo
git clone https://github.com/avihu28/Quantum-Safe-Bitcoin-Transactions.git
cd Quantum-Safe-Bitcoin-Transactions

# 4. Build GPU kernels
cd gpu
make
cd ..

# 5. Python environment
cd pipeline
python3 -m venv venv
source venv/bin/activate
pip install coincurve  # main dep; add others if import errors appear (requests, etc.)

# Test
python3 qsb_pipeline.py --help
```

**Windows (via WSL2 – Strongly Recommended)**
QSB’s Makefile/CUDA build is Linux-first. Native Windows CUDA is possible but painful.

1. Enable **WSL2** + Ubuntu 24.04 (or any distro) from Microsoft Store.
2. Inside WSL Ubuntu terminal, run the **exact same Linux commands** above.
3. Share your Windows Bitcoin data dir with WSL if needed (`/mnt/c/Users/...`).
4. GPU passthrough in WSL2 works perfectly with NVIDIA CUDA (install CUDA inside WSL).

**Native Windows (Advanced / Not Recommended)**
- Install **CUDA Toolkit** for Windows + Visual Studio Build Tools.
- Manually convert the Makefile to MSBuild / nmake (not officially supported).
- Use the same Python steps. Most users should just use WSL2.

### Phase 3: Full Quantum-Safe Spending Workflow (All Platforms)
Once everything is installed:

1. **Fund the QSB output** (use btcwallet or btcctl to send to a special bare-script address generated by the pipeline).
2. Run the pipeline (example from repo):
   ```bash
   cd pipeline
   # Setup (generates keys/script)
   python3 qsb_pipeline.py setup --config A

   # After funding
   python3 qsb_pipeline.py export --funding-txid <TXID> --funding-vout 0 --funding-value <SATS> --dest-address <PUBKEYHASH>

   # GPU search (pinning)
   cd ../gpu
   ./launch_multi_gpu.sh pinning

   # After pinning hit → digest rounds
   ./launch_multi_gpu.sh digest ../digest_r1.bin
   ./launch_multi_gpu.sh digest ../digest_r2.bin

   # Assemble final tx
   cd ../pipeline
   python3 qsb_pipeline.py assemble --locktime <LT> --sequence <SEQ> --round1 <INDICES> --round2 <INDICES> ...
   ```
3. **Broadcast** the resulting raw transaction:
   - Use `btcctl` (from btcwallet) or a direct RPC call to btcd, **or**
   - Submit via Slipstream / a miner’s private endpoint (required for non-standard tx).

**Monitoring**: Check `results/pinning_hit.txt` and GPU logs.

### Phase 4: Integration Tips with btcd + btcwallet
- Keep **btcd** running as your always-on node (it validates QSB txs perfectly).
- Use **btcwallet** normally for daily ECDSA spends.
- For automation: Write a small Go wrapper (or Python script) that calls `btcctl` / btcd RPC after the QSB pipeline finishes.
- Store the QSB-generated script template in your btcwallet data dir for reuse.
